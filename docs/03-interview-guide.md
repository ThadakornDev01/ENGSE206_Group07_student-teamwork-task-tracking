# 03 — Interview Guide and Rehearsal Revision

> Team: Group 07 | Case: 07 — ระบบติดตามงานกลุ่มและการแบ่งบทบาทสมาชิก | Target role: หัวหน้ากลุ่ม, สมาชิกกลุ่ม, อาจารย์ผู้สอน, เพื่อน Reviewer

## 1. Opening and Consent Script

"สวัสดีครับ พวกเราจากกลุ่ม 07 กำลังทำโครงงานออกแบบ 'ระบบติดตามงานกลุ่มและการแบ่งบทบาทสมาชิก' วันนี้จึงขอรบกวนเวลาสัมภาษณ์ประมาณ 15-20 นาที เพื่อสอบถามมุมมองและประสบการณ์ในการทำงานกลุ่มครับ ข้อมูลที่ได้จะใช้สำหรับออกแบบระบบในวิชา ENGSE206 เท่านั้น จะไม่มีการบันทึกข้อมูลส่วนบุคคล หรือนำไปเปิดเผยต่อสาธารณะ หากมีคำถามใดที่ท่านไม่สะดวกตอบสามารถแจ้งข้ามได้ทันทีครับ"

## 2. Interview Questions

| Q ID | EO ID | Main / Follow-up Question | Type | Why / Expected Evidence |
|---|---|---|---|---|
| Q-01 | EO-02 | [หัวหน้ากลุ่ม] ปกติเวลาสั่งงานและตามงานเพื่อนในกลุ่มผ่านแชต มีขั้นตอนอย่างไร และพบปัญหาอะไรบ่อยที่สุด? | Open | เพื่อเข้าใจ Workflow ปัจจุบัน และปัญหาหลักในการติดตามความคืบหน้าของงาน |
| Q-02 | EO-02 | [หัวหน้ากลุ่ม] จากปัญหาที่บอกมา เวลาเพื่อนส่งงาน ข้อมูลหรือหลักฐานรูปแบบใดที่คุณคิดว่า "ครบถ้วน" และช่วยให้คุณประเมินได้ทันทีว่างานนั้นเสร็จจริง? | Open / Probe | เพื่อค้นหา Requirement เรื่องประเภทของหลักฐานการทำงาน (Evidence) ที่ต้องมีในระบบ |
| Q-03 | EO-03 | [หัวหน้ากลุ่ม] หากมีกรณีที่สมาชิกทำงานไม่ไหว หรือหายไป แล้วต้องเปลี่ยนคนรับผิดชอบกลางคัน ปกติมีวิธีการจัดการและส่งมอบงานกันอย่างไร? | Open / Exception | เพื่อค้นหากระบวนการรับมือกับปัญหาเฉพาะหน้า และหาความจำเป็นของระบบ History |

| Q-05 | EO-02 | [สมาชิกกลุ่ม] เวลาที่ได้รับมอบหมายงาน ปกติคุณอยากให้หัวหน้า หรือระบบ แจ้งข้อมูลอะไรบ้าง เพื่อให้สามารถเริ่มงานได้ทันทีโดยไม่ต้องถามซ้ำ? | Open | เพื่อหา Data Requirement พื้นฐานที่ระบบต้องแสดงผลในหน้า Task ของแต่ละคน |
| Q-06 | EO-02 | [สมาชิกกลุ่ม] ตอนที่ทำงานเสร็จแล้วและต้องส่งหลักฐานให้เพื่อนดู มีความกังวลหรืออุปสรรคอะไรไหมในการเตรียมหลักฐาน (เช่น ไฟล์ใหญ่ไป ส่งลิงก์ผิด)? | Open / Probe | เพื่อหา Constraint หรือข้อจำกัดของการส่งหลักฐานการทำงาน |
| Q-07 | EO-03 | [สมาชิกกลุ่ม] เคยมีกรณีที่ต้องทำต่องานจากเพื่อนไหม? ตอนนั้นรู้ได้อย่างไรว่าเพื่อนทำถึงไหนแล้ว และข้อมูลอะไรที่มักจะขาดหายไปตอนรับช่วงต่อ? | Open / Exception | เพื่อยืนยันความจำเป็นของข้อมูลประวัติการทำงาน (History Log) จากมุมมองของผู้รับช่วงต่อ |
| Q-08 | EO-01 | [อาจารย์ผู้สอน] เวลาตรวจงานกลุ่มของนักศึกษา ปกติอาจารย์ประเมินคะแนนการมีส่วนร่วมของแต่ละคนจากเกณฑ์หรือข้อมูลส่วนไหนบ้างครับ? | Open | เพื่อเข้าใจเกณฑ์การให้คะแนน และเป้าหมายสูงสุดในการใช้ข้อมูลของผู้สอน |
| Q-09 | EO-01 | [อาจารย์ผู้สอน] หากมีระบบที่สรุปข้อมูลมาให้ อาจารย์ต้องการเห็นข้อมูลอะไรเป็นอันดับแรกในหน้า Dashboard เพื่อให้สามารถตัดสินใจให้คะแนนได้ทันที? | Open / Probe | เพื่อค้นหา Requirement เชิงลึกสำหรับการออกแบบหน้า Dashboard ให้ตรงใจผู้ใช้งาน |

| Q-11 | EO-01 | [เพื่อน Reviewer] ตอนที่ต้องให้คะแนนเพื่อนในกลุ่ม (Peer Evaluation) ปกติใช้เกณฑ์อะไรในการตัดสินใจว่าเพื่อนคนไหนควรได้คะแนนเท่าไหร่? | Open | เพื่อเข้าใจ Workflow และความลำบากใจในการประเมินเพื่อนร่วมทีม |

## 3. Bias and Privacy Check

| Q ID | Issue: leading/solution/double/vague/privacy | Neutral revision | Rationale |
|---|---|---|---|
| Q-09 | (Leading/Solution) เดิมทีทีมอาจถามว่า "อาจารย์อยากได้หน้า Dashboard ที่มีกราฟแสดงผลแบบแท่งไหมครับ?" | ปรับเป็น: "อาจารย์ต้องการเห็นข้อมูลอะไรเป็นอันดับแรกในหน้า Dashboard..." | ไม่ชี้นำให้ตอบตามรูปแบบ (Solution) ที่ทีมคิดไว้ล่วงหน้า เปิดกว้างให้เสนอข้อมูลที่ต้องการจริงๆ |
| Q-04 | (Vague) คำว่า "เปลี่ยนตัวคนทำ" อาจจะกว้างไป | ปรับเป็น: "ตอนที่มีการเปลี่ยนคนทำ จำเป็นต้องเก็บประวัติไหมว่าคนเก่าทำอะไรไปบ้าง..." | เพื่อกำหนด Scope ให้ชัดเจนว่าหมายถึงประวัติของเนื้องานนั้นๆ และหลีกเลี่ยงการพาดพิงเรื่องส่วนตัว |

## 4. AI Interview Rehearsal

- **Role/prompt used:** สวมบทบาทเป็น "หัวหน้ากลุ่ม" และ "อาจารย์ผู้สอน" ใน Case 07 ตอบคำถามโดยอิงจากปัญหาการสั่งงานและตรวจงานผ่านแชต
- **Session/evidence path:** จำลองการถามตั้งแต่ Q-01 ถึง Q-13 กับ AI
- **Simulation limitation:** AI ขาดประสบการณ์จริงในการเผชิญหน้ากับความขัดแย้งของเพื่อนร่วมทีม (Conflict) ทำให้คำตอบอาจดูเป็นอุดมคติเกินไป

| Q ID | What happened | Useful probe / problem | Revised question |
|---|---|---|---|
| Q-01 | AI ตอบกว้างมาก บอกเพียงว่า "ตามงานในไลน์แล้วเพื่อนไม่ตอบ" | คำตอบไม่บอกถึงกระบวนการแก้ไขปัญหาของหัวหน้ากลุ่ม | (เพิ่ม Probe ในตอนสัมภาษณ์จริง): "แล้วถ้าเพื่อนไม่ตอบ หรือบอกว่ายังไม่เสร็จ ปกติคุณมีวิธีตามงานต่ออย่างไรครับ?" |
